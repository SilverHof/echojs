Экосистема развивается итеративно: часть пакетов уже имеет workspace-реализацию, часть — пока как “контур” (папка + README-заметка).

## Активные пакеты (workspace)

- [`@echojs/http`](/packages/http/) — type-safe HTTP client (hooks, adapters, strong errors).
- [`@echojs/reactivity`](/packages/reactivity/) — строгая реактивность поверх engine, с собственным surface API.

## Планируемые пакеты (дизайн/заготовки)

Эти пакеты сейчас присутствуют как директории в `packages/`, но ещё не оформлены как publishable `@echojs/*` workspace-пакеты:

- `echojs-router`

## Как добавить новый пакет в экосистему

Коротко: добавь `packages/<dir>/package.json` с `name: "@echojs/<slug>"`, публичный entrypoint `src/index.ts`, и документацию в `/packages/<slug>/`.
