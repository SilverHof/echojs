EchoJS — это экосистема пакетов `@echojs/*`, которые вместе образуют мой фреймворк.

## Что здесь важно понимать

- **Экосистема > монолит**: каждый пакет решает свою задачу и может использоваться отдельно.
- **Exports — это контракт**: публичный API определяется тем, что реально экспортируется из пакета.
- **Документация двуслойная**: ручные гайды + автогенерируемый API reference.

## Быстрый выбор пакета

- Нужен HTTP клиент с hooks/adapters и сильной моделью ошибок → [`@echojs/http`](/packages/http/).
- Нужна строгая реактивность как фундамент UI → [`@echojs/reactivity`](/packages/reactivity/).

## Как читать документацию

1) **Overview** — что это за пакет и какие задачи решает.
2) **Quick Start** — минимальный “рабочий” пример.
3) **Concepts / Guides** — контракты и важные нюансы.
4) **API Reference** — подробный справочник по экспортам.

EchoJS — это монорепо и экосистема пакетов `@echojs/*`.

## Быстрый старт

1) Выберите пакет в разделе **Packages**.

2) Начните с Quick Start конкретного пакета.

## Первые шаги

- Если вам нужен HTTP-клиент: начните с [`@echojs/http`](/packages/http/).
- Если нужна реактивность: начните с [`@echojs/reactivity`](/packages/reactivity/).
