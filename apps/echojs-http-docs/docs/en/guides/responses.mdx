In `@echojs/http`, a response is an `HttpResponse` object with body readers and convenience helpers.

## JSON / text / bytes

```ts
type User = { id: string; name: string };

const http = createHttpClient({ baseUrl: "https://api.example.com" });

const users = await http.get("/users").json<User[]>();
const robots = await http.get("/robots.txt").text();
const logo = await http.get("/favicon.ico").bytes();
```

## unwrapJson()

`unwrapJson<T>()` is sugar for the common case: parse JSON and assume `ok`.

```ts
const me = await http.get("/me").unwrapJson<{ id: string }>();
```

## raw() — return Response even on 4xx/5xx

```ts
const res = await http.raw({ url: "/maybe-missing" });
if (!res.ok) {
  console.warn(res.status, await res.text());
}
```

## stream() — if adapter supports it

```ts
const res = await http.stream({ url: "/big.csv" });
// res.body can be a ReadableStream in browsers / compatible runtimes
```
